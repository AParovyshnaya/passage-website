How to customize licensing colors for product.

Eclipse Passage allows you to configure the color schema for licensing restriction level representation.
The licensing status dialog displays product components to be licensed.

The level of licensing restrictions are

  • INFO - inform about functionality restriction without pausing the execution flow;
  • WARNING - pause the execution flow with warning, but allow to proceed without functionality blocking;
  • ERROR - interrupt the execution for the restricted functionality with error, but allow to use other scenarios;
  • FATAL - terminate the execution with fatal error;

Particular level refers to own color scheme defined for the specific product. Licensing restrictions color can be configured from settings dialog or using preferenceCustomization.

To customize the licensing color scheme by following steps:

  • specify the file containing default preference values for the product via extension point org.eclipse.core.runtime.products by adding:
< property
			name="preferenceCustomization"
			value="platform:/<bundle>/plugin_customization.ini"/>

Add lines to file “plugin_customization.ini” file with next format:

org.eclipse.passage.lic.jface/COLOR_LEVEL_OK = RGB { 100 , 200, 255}
org.eclipse.passage.lic.jface/COLOR_LEVEL_INFO = RGB { 120, 255, 255}
org.eclipse.passage.lic.jface/COLOR_LEVEL_WARN = RGB { 120, 255, 255}
org.eclipse.passage.lic.jface/COLOR_LEVEL_ERROR = RGB { 130 , 255, 255}
org.eclipse.passage.lic.jface/COLOR_LEVEL_FATAL = RGB { 130 , 255, 255}

The color specification represent in terms of the primary additive color model (red, green and blue):

- RGB {red,green,blue}
